term: V8
short: Google's JavaScript engine (C++), the thing that actually parses, compiles, and runs JS. It powers Chrome and Node. The runtime "computer" for JavaScript, analogous to the CPython VM for Python.
domain: js
related: node, browser, bytecode, cpython, python-vm

V8 is the JavaScript engine built by Google, written in C++. It's the
component that takes JavaScript source and runs it: parse → compile to
[[bytecode]] (its **Ignition** interpreter) → JIT-compile hot code to
native machine code (**TurboFan**). It also owns the garbage collector and
the single-threaded [[event-loop|event loop]]'s execution side.

V8 is not a language and not a runtime *environment* — it's the engine
embedded inside one. The two main embeddings:

- [[browser|The browser]] (Chrome and Chromium-based) — V8 + DOM APIs.
- [[node|Node.js]] — V8 + filesystem/network APIs.

For a Python developer the clean mapping is: **V8 is to JavaScript what
[[cpython|CPython]] (and its [[python-vm|VM]]) is to Python** — the
concrete program that executes a language people otherwise discuss
abstractly.

This matters to TypeScript because TS never runs as TS: it's
[[type-erasure|erased]] to JavaScript, and *that* is what V8 executes. So
"will this run?" is always a question about the JavaScript V8 sees, never
about the types.
